План работ (локально)

Sprint 1 — Принимаемый минимум (must-have)

Цель: чтобы XML для ON_AKTREZRABP был не только XSD-valid, но и близок к “принимаемому” по бизнес-правилам.

1. ИдФайл по регламенту

• конструктор из частей (операторы/коды/дата/GUID)
• контроль совпадения ИдФайл и имени файла
• read-only + advanced override

2. НастрФормДок как управляющий блок

• ПрНДСВИтог, ПрНакИтог, ПрИндЦен, ПрСведРасчСогл, СтепАгрег
• на их основе включать/делать обязательными нужные поля

3. Rule-engine v1

• условная обязательность сверх XSD
• понятные ошибки “какое условие нарушено”

4. Верх документа

• зафиксировать КНД=1110335, ВерсФорм=1.00
• проверить обязательные атрибуты Документ

Результат спринта: стабильный “базовый рабочий контур” для P-файла.

───

Sprint 2 — Строки и итоги (расчётный контур)

Цель: закрыть основную практику заполнения КС-2/КС-3.

1. Режимы НаимИСт

• ВидРаб / Раздел / РасшифРес

2. Строка ВидРаб

• добавить ТипЗатр
• корректный ОКЕИ_Стройка
• логика единиц (0000, НаимЕдИзм, ограничения)

3. Итоги ВсегоАктОтч

• полный набор полей (без НДС / НДС / с НДС / ставка)
• режимы НДС: “по строкам” vs “в итоге”

Результат спринта: типовой реальный документ собирается без ручной правки XML.
