# Проверка соответствия репозитория требованиям `nalog target.docx`

Дата проверки: 2026-03-08.

## Что проверялось

1. Требования из `nalog target.docx` (извлечены из `word/document.xml`):
   - формат имени файла `ON_AKTREZRABP_A_O_YYYYMMDD_N`;
   - фиксированные значения версии формата `1.00` (часть 971) и КНД;
   - обязательные блоки верхнего уровня документа;
   - управляющие флаги `НастрФормДок`;
   - зависимые блоки (`СвОРасч`, `ВсегоАктСНач`, НДС-режимы, валюта договора).
2. Реализация в коде приложения (`app/main.py`, `app/templates/index.html`, `README.md`).
3. Фактическая проверка сценариев через `scripts/check_scenarios.py` (с XSD и без XSD).

## Результат по ключевым требованиям

| Требование из формата | Статус в репозитории | Подтверждение |
|---|---|---|
| `ИдФайл` по шаблону `ON_AKTREZRABP_A_O_YYYYMMDD_GUID` | Частично соответствует | В коде есть автогенерация и regex-проверка шаблона; доступен ручной режим. |
| `ВерсФорм = 1.00`, `КНД = 1110335` фиксированы | Соответствует | Значения принудительно проставляются в `generate`, отображаются read-only в UI. |
| Наличие блока `НастрФормДок` (ПрНДСВИтог, ПрНакИтог, ПрИндЦен, ПрСведРасчСогл, СтепАгрег) | Частично соответствует | Поля выведены в UI и участвуют в проверках условной обязательности/комбинаций. |
| Условная обязательность `СвОРасч`, `ВсегоАктСНач`, НДС-ветки, валютные поля | Частично соответствует | Есть rule-engine `_conditional_required` и `_impossible_combinations`. |
| Общая XSD-совместимость формируемых сценариев | Не соответствует | `python scripts/check_scenarios.py --xsd` даёт 7/7 FAIL (критичные пропуски и порядок элементов). |

## Критичные расхождения, найденные проверкой

При запуске `python scripts/check_scenarios.py --xsd` для всех 7 эталонных сценариев получены одинаковые ошибки XSD:

- отсутствует обязательный атрибут `@ВерсПрог` у корневого узла `Файл`;
- отсутствуют обязательные атрибуты `@ДатаИнфПодр`, `@ВремИнфПодр`, `@НаимЭкСубСост` у `Документ`;
- нарушен порядок/структура: `ВсегоАктОтч` появляется раньше обязательного `СвАктСдПр`.

Это означает, что внутренние сценарии проходят только локальные бизнес-правила (`check_scenarios.py` без `--xsd`), но не проходят итоговую проверку по актуальной XSD.

## Итоговый вывод

Репозиторий **частично соответствует** требованиям `nalog target.docx`: реализованы важные элементы (конструктор `ИдФайл`, фиксированные системные поля, блок `НастрФормДок`, условные правила), но на уровне строгого соответствия XSD есть критичные несоответствия.

Текущий статус: **готовность для реального обмена по формату ФНС не достигнута** до устранения XSD-ошибок (обязательные атрибуты + корректный порядок элементов).
