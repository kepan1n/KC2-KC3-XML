# KC2-KC3-XML

Веб-приложение для подготовки **КС-2/КС-3** и генерации XML по требованиям ФНС (формат `ON_AKTREZRABP`).

Основание: материалы в `nalog docs/` (аудит, спринты, целевой документ, актуальная XSD).

---

## Цель проекта

Сделать удобный браузерный инструмент, который:

1. собирает данные КС-2/КС-3 в понятной форме,
2. применяет правила обязательности полей по логике формата,
3. генерирует XML, соответствующий XSD ФНС,
4. помогает избежать ручной правки XML и типичных ошибок перед отправкой.

---

## Актуальные файлы требований (источник истины)

`nalog docs/`:

- `ON_AKTREZRABP_1_971_01_01_00_03.xsd` — актуальная схема
- `audit nalog.txt` — gap-анализ и приоритеты
- `sprints.txt` — план работ по этапам
- `nalog target.docx` — целевое описание и контекст требований

---

## Что уже реализовано

- Парсинг XSD и построение формы по структуре схемы
- Маркировка обязательных/опциональных полей
- Генерация XML (`windows-1251`) и локальная XSD-валидация
- Профили и сохранение реквизитов (`data/defaults.json`, `data/profiles`)
- Подсказки по полям в интерфейсе (`?`)
- Блок `НастрФормДок` в верхней панели
- Базовый rule-engine условной обязательности
- Конструктор `ИдФайл` + проверка формата
- Rule-engine v2: условно-обязательные поля + проверки невозможных комбинаций флагов
- Зафиксированные системные поля:
  - `ВерсФорм = 1.00`
  - `КНД = 1110335`
- Эталонные сценарии Sprint A: `examples/scenarios/*.json`

## Roadmap

### Sprint A — Принимаемый минимум

- Довести rule-engine до стабильного v2
- Закрыть критичные условия от `НастрФормДок`
- Финализировать проверки верхнего уровня `Документ`
- Собрать набор эталонных сценариев (минимум валидных кейсов)

### Sprint B — Строки и итоги

- Режимы `НаимИСт`: `ВидРаб / Раздел / РасшифРес`
- Расширение строки `ВидРаб` (`ТипЗатр`, `ОКЕИ_Стройка`, единицы)
- Полный блок `ВсегоАктОтч`
- Корректные режимы НДС: по строкам / в итоге

### Sprint C — Продвинутый контур P

- `ОсновСтроит`
- `СвОРасч` (расширенный)
- `ВсегоАктСНач` (полный)
- `Подписант` + доверенности (МЧД/бумажная)
- `ИспрАктСдПр`

### Sprint D — Контур Z

- Мастер формирования `ON_AKTREZRABZ`
- Маппинг из P в Z
- Проверка согласованности пары файлов

## Запуск

```bash
cd /home/konstantin/KC2-KC3-XML
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8080
```

Открыть в браузере: `http://localhost:8080`

## Важно
